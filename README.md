Возможно наличие неоптимизированного кода, дублирующегося кода, отсутствие обработки некоторых ошибок и вообще код кривой, но оно работает :) 

В скором времени я исправлю недочеты, напишу четкий Readme, структурирую код, обработаю ошибки, составлю единый пайплайн, попробую адаптировать алгоритм для решения не только определнного тестового задания определенной компании, а сделать его полезным обществу :)  

## Как работает алгоритм. Анализ решений (на примере 100)

1) Получить все форки
2) Склонировать репозитории
3) закинуть в ИИ решение (100 запросов), каждое решение отправляется отдельно и изолированно от других (системный промпт лежит по адресу prompts/system_prompt.txt)
4) Грубая фильтрация: выбрать работы с оценкой ['отлично']
5) Методом Турнир + Losers bracket выявить лучшую работу. Точность 90%. (Можно находить не только лучшую работу, но и составить свой рейтинг топ-4) (системный промпт лежит по адресу tournament_prompts/system_prompt.txt)

промпт текстового задания: prompts/exercise_text.txt
