# Task Service

Сервис для управления задачами с HTTP API на Go.

## Требования

- Go `1.23+`
- Docker и Docker Compose

## Быстрый запуск через Docker Compose

```bash
docker compose up --build
```

После запуска сервис будет доступен по адресу `http://localhost:8080`.

Если `postgres` уже запускался ранее со старой схемой, пересоздай volume:

```bash
docker compose down -v
docker compose up --build
```

Причина в том, что SQL-файлы из `migrations/` монтируются в `docker-entrypoint-initdb.d` и применяются только при инициализации пустого data volume.

## Swagger

Swagger UI:

```text
http://localhost:8080/swagger/
```

OpenAPI JSON:

```text
http://localhost:8080/swagger/openapi.json
```

## API

Базовый префикс API:

```text
/api/v1
```

Обычные задачи:

- `POST   /api/v1/tasks`
- `POST /api/v1/tasks/batch`
- `GET    /api/v1/tasks`
- `GET    /api/v1/tasks/{id}`
- `PUT    /api/v1/tasks/{id}`
- `DELETE /api/v1/tasks/{id}`

Шаблоны периодических задач:

- `POST   /api/v1/task-templates`
- `GET    /api/v1/task-templates`
- `GET    /api/v1/task-templates/{id}`
- `PUT    /api/v1/task-templates/{id}`
- `DELETE /api/v1/task-templates/{id}`

## Реализация периодических задач

### Общий подход

Периодические задачи реализованы с помощью шаблона `task_templates`. Шаблон описывает расписание, а конкретные задачи генерируются на его основе и хранятся в той же таблице `tasks`.
Данный подход позволяет не менять существующий API задач, и врач может работать с периодическими задачами так же как с обычными (менять статус, изменять, удалять через стандартный API)

### Схема БД

`tasks`
- `id` - Первичный ключ
- `title` - Название задачи
- `description` - Описание задачи
- `status` - Статус: `new`, `in_progress`, `done`
- `template_id` - Ссылка на шаблон. `NULL` для обычных задач. При удалении шаблона обнуляется автоматически (`ON DELETE SET NULL`)
- `scheduled_for` - Дата на которую создана задача. `NULL` для обычных задач
- `created_at` - Дата создания записи
- `updated_at` - Дата последнего обновления записи

`task_templates`

- `id` - Первичный ключ
- `title` - Название задачи
- `description` - Описание задачи
- `schedule_type` - Тип расписания
- `every_n_days` - Шаг в днях. Только для `daily`
- `day_of_month` - Число месяца от 1 до 30. Только для `monthly_day`
- `start_date` - Дата начала генерации задач
- `created_at` - Дата создания шаблона
- `updated_at` - Дата последнего обновления шаблона

Таблица `task_templates` содержит поля расписания с CHECK constraints на уровне БД:
- для `daily` - `every_n_days` обязателен и >= 1;
- для `monthly_day` - `day_of_month` обязателен от 1 до 30;
- для остальных типов - лишние поля должны быть `NULL`, чтобы не было мусорных данных и последующих ошибок из-за них.

**Примечание:** в Swagger пример ответа `TaskTemplate` показывает оба поля `every_n_days` и `day_of_month` заполненными одновременно. На практике только одно из них будет заполнено в зависимости от `schedule_type`, второе будет отсутствовать в ответе.

### Типы расписаний

- `daily` — каждые N дней начиная с `start_date`. Требует `every_n_days >= 1`.
- `monthly_day` — каждое конкретное число месяца от 1 до 30. Если в месяце нет такого числа (например, 30 февраля), тогда месяц пропускается.
- `even_days` — только чётные числа месяца.
- `odd_days` — только нечётные числа месяца.

### Задачи на конкретные даты

Для создания задач на конкретные даты используется эндпоинт `POST /api/v1/tasks/batch`.
Он принимает список дат и создаёт отдельную задачу на каждую из них.

Этот подход выбран вместо отдельного типа расписания в шаблонах
потому что задачи на конкретные даты одноразовые и не имеют
периодичности. Тащить их в механизм шаблонов было бы неверно.

Созданные задачи на конкретные даты - это обычные задачи без `template_id` и `scheduled_for`.
### Генерация задач

При создании шаблона сразу генерируются задачи на 30 дней вперёд от `start_date`. 30 дней выбрано чтобы было достаточно для планирования и не перегружало БД при создании. При необходимости количество дней можно легко изменить.

При обновлении шаблона все будущие задачи со статусом `new` удаляются, затем генерируются заново по новому расписанию. Было выбрано удалить задачи и создать новые, потому что врач может поменять с четных на нечетные дни, либо выставить другой день месяца, тогда изменение задач не имело бы смысла и все равно пришлось бы создавать новые.  

Задачи в статусе `in_progress` не трогаются, так как уже были взяты в работу. Но при необходимости врач может легко их изменить или удалить через стандартный API.

При удалении шаблона сначала удаляются будущие `new` задачи, затем удаляется сам шаблон. У оставшихся задач (`in_progress`, `done`) поле `template_id` автоматически обнуляется через `ON DELETE SET NULL`. Это сделано чтобы история была сохранена.

Обновление и удаление выполняются в транзакции, поэтому при сбое системы не случится ситуации, когда старые задачи будут удалены, а новые не добавлены. Для этого реализован интерфейс `Transactor` с методом `WithTx`  

### Защита от дублей

В таблице `tasks` есть уникальный индекс по `(template_id, scheduled_for)`. При попытке вставить уже существующую задачу используется `ON CONFLICT DO NOTHING`. Это позволяет безопасно перезапускать генерацию без риска дублей.

### Последующая генерация шаблонных задач

Запускается горутина при старте сервиса и срабатывает раз в 24 часа. При каждом запуске проходит по всем шаблонам и генерирует новые задачи на 30 дней вперед. Таким образом у шаблонов всегда есть задачи на ближайший месяц вперёд.

Горутина останавливается вместе с сервисом через общий контекст.

### Примеры запросов

**Создать задачи на конкретные даты:**
```json
POST /api/v1/tasks/batch
{
  "title": "Patient call",
  "description": "Call all patients on the list",
  "dates": ["2026-04-20", "2026-05-01"]
}
```

**Создать шаблон `daily`:**
```json
POST /api/v1/task-templates
{
  "title": "Daily patient call",
  "schedule_type": "daily",
  "every_n_days": 1,
  "start_date": "2026-04-10"
}
```

**Создать шаблон `monthly_day`:**
```json
POST /api/v1/task-templates
{
  "title": "Monthly report",
  "schedule_type": "monthly_day",
  "day_of_month": 15,
  "start_date": "2026-04-10"
}
```

**Создать шаблон `even_days` (для `odd_days` аналагично):**
```json
POST /api/v1/task-templates
{
  "title": "Inventory check",
  "schedule_type": "even_days",
  "start_date": "2026-04-10"
}
```

package main

import (
	"context"
	"fmt"
	"log/slog"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"

	infrastructurepostgres "example.com/taskservice/internal/infrastructure/postgres"
	postgresrepo "example.com/taskservice/internal/repository/postgres"
	"example.com/taskservice/internal/scheduler"
	transporthttp "example.com/taskservice/internal/transport/http"
	swaggerdocs "example.com/taskservice/internal/transport/http/docs"
	httphandlers "example.com/taskservice/internal/transport/http/handlers"
	"example.com/taskservice/internal/usecase/task"
)

func main() {
	logger := slog.New(slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{
		Level: slog.LevelInfo,
	}))

	cfg := loadConfig()

	ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
	defer stop()

	pool, err := infrastructurepostgres.Open(ctx, cfg.DatabaseDSN)
	if err != nil {
		logger.Error("open postgres", "error", err)
		os.Exit(1)
	}
	defer pool.Close()

	taskRepo := postgresrepo.New(pool)
	templateRepo := postgresrepo.NewTemplateRepository(pool)

	transactor := postgresrepo.NewTransactor(pool)
	templateUsecase := task.NewTemplateService(templateRepo, taskRepo, transactor)
	taskUsecase := task.NewService(taskRepo)

	taskHandler := httphandlers.NewTaskHandler(taskUsecase)
	templateHandler := httphandlers.NewTemplateHandler(templateUsecase)
	docsHandler := swaggerdocs.NewHandler()

	router := transporthttp.NewRouter(taskHandler, templateHandler, docsHandler)

	schedule := scheduler.NewScheduler(templateUsecase, 24*time.Hour, logger)
	go schedule.Run(ctx)

	server := &http.Server{
		Addr:              cfg.HTTPAddr,
		Handler:           router,
		ReadHeaderTimeout: 5 * time.Second,
	}

	go func() {
		<-ctx.Done()

		shutdownCtx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
		defer cancel()

		if err := server.Shutdown(shutdownCtx); err != nil {
			logger.Error("shutdown http server", "error", err)
		}
	}()

	logger.Info("http server started", "addr", cfg.HTTPAddr)

	if err := server.ListenAndServe(); err != nil && err != http.ErrServerClosed {
		logger.Error("listen and serve", "error", err)
		os.Exit(1)
	}
}

type config struct {
	HTTPAddr    string
	DatabaseDSN string
}

func loadConfig() config {
	cfg := config{
		HTTPAddr:    envOrDefault("HTTP_ADDR", ":8080"),
		DatabaseDSN: envOrDefault("DATABASE_DSN", "postgres://postgres:postgres@localhost:5432/taskservice?sslmode=disable"),
	}

	if cfg.DatabaseDSN == "" {
		panic(fmt.Errorf("DATABASE_DSN is required"))
	}

	return cfg
}

func envOrDefault(key, fallback string) string {
	if value := os.Getenv(key); value != "" {
		return value
	}

	return fallback
}

type Status string

const (
	StatusNew        Status = "new"
	StatusInProgress Status = "in_progress"
	StatusDone       Status = "done"
)

type Task struct {
	ID           int64      `json:"id"`
	Title        string     `json:"title"`
	Description  string     `json:"description"`
	Status       Status     `json:"status"`
	TemplateID   *int64     `json:"template_id,omitempty"`
	ScheduledFor *time.Time `json:"scheduled_for,omitempty"`
	CreatedAt    time.Time  `json:"created_at"`
	UpdatedAt    time.Time  `json:"updated_at"`
}

func (s Status) Valid() bool {
	switch s {
	case StatusNew, StatusInProgress, StatusDone:
		return true
	default:
		return false
	}
}

package postgres

import (
	"context"
	"fmt"

	"github.com/jackc/pgx/v5/pgxpool"
)

func Open(ctx context.Context, dsn string) (*pgxpool.Pool, error) {
	if dsn == "" {
		return nil, fmt.Errorf("database dsn is empty")
	}

	cfg, err := pgxpool.ParseConfig(dsn)
	if err != nil {
		return nil, err
	}

	pool, err := pgxpool.NewWithConfig(ctx, cfg)
	if err != nil {
		return nil, err

package scheduler

import (
	"context"
	"log/slog"
	"time"

	"example.com/taskservice/internal/usecase/task"
)

type Scheduler struct {
	templateUsecase task.TemplateUsecase
	interval        time.Duration
	logger          *slog.Logger
}

func NewScheduler(templateUsecase task.TemplateUsecase, interval time.Duration, logger *slog.Logger) *Scheduler {
	return &Scheduler{
		templateUsecase: templateUsecase,
		interval:        interval,
		logger:          logger,
	}
}

func (s Scheduler) Run(ctx context.Context) {
	err := s.templateUsecase.RefreshTasks(ctx)
	if err != nil {
		slog.Error("failed to refresh tasks")
	}

	ticker := time.NewTicker(s.interval)
	defer ticker.Stop()

	for {
		select {
		case <-ticker.C:
			err = s.templateUsecase.RefreshTasks(ctx)
			if err != nil {
				slog.Error("failed to refresh tasks")
			}
		case <-ctx.Done():
			slog.Info("scheduler stopped")
			return
		}
	}
}

func (h *TaskHandler) BatchCreateByDates(w http.ResponseWriter, r *http.Request) {
	var req batchCreateByDatesDTO
	if err := decodeJSON(r, &req); err != nil {
		writeError(w, http.StatusBadRequest, err)
		return
	}

	if len(req.Dates) == 0 {
		writeError(w, http.StatusBadRequest, errors.New("dates must not be empty"))
		return
	}

	dates := make([]time.Time, 0, len(req.Dates))
	for _, d := range req.Dates {
		parsed, err := time.Parse("2006-01-02", d)
		if err != nil {
			writeError(w, http.StatusBadRequest, errors.New("invalid date format"))
			return
		}
		dates = append(dates, parsed)
	}

	created, err := h.usecase.BatchCreateByDates(r.Context(), taskusecase.BatchCreateByDatesInput{
		Title:       req.Title,
		Description: req.Description,
		Dates:       dates,
	})
	if err != nil {
		writeUsecaseError(w, err)
		return
	}

	response := make([]taskDTO, 0, len(created))
	for i := range created {
		response = append(response, newTaskDTO(&created[i]))
	}

	writeJSON(w, http.StatusCreated, response)

func getIDFromRequest(r *http.Request) (int64, error) {
	rawID := mux.Vars(r)["id"]
	if rawID == "" {
		return 0, errors.New("missing task id")
	}

	id, err := strconv.ParseInt(rawID, 10, 64)
	if err != nil {
		return 0, errors.New("invalid task id")
	}

	if id <= 0 {
		return 0, errors.New("invalid task id")
	}

	return id, nil
}

func decodeJSON(r *http.Request, dst any) error {
	decoder := json.NewDecoder(r.Body)
	decoder.DisallowUnknownFields()

	if err := decoder.Decode(dst); err != nil {
		return err
	}

	return nil
}

func writeUsecaseError(w http.ResponseWriter, err error) {
	switch {
	case errors.Is(err, taskdomain.ErrNotFound):
		writeError(w, http.StatusNotFound, err)
	case errors.Is(err, taskusecase.ErrInvalidInput):
		writeError(w, http.StatusBadRequest, err)
	default:
		writeError(w, http.StatusInternalServerError, err)
	}
}

func writeError(w http.ResponseWriter, status int, err error) {
	writeJSON(w, status, map[string]string{
		"error": err.Error(),
	})
}

	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(status)

	_ = json.NewEncoder(w).Encode(payload)
}

		return nil
	}
}

func generateDaily(startDate time.Time, everyNDays int, from time.Time) []time.Time {
	until := from.AddDate(0, 0, generationHorizonDays)

	first := startDate
	for first.Before(from) {
		first = first.AddDate(0, 0, everyNDays)
	}

	var dates []time.Time
	for day := first; !day.After(until); day = day.AddDate(0, 0, everyNDays) {
		dates = append(dates, day)
	}

	return dates
}

func generateMonthlyDay(dayOfMonth int, from time.Time) []time.Time {
	until := from.AddDate(0, 0, generationHorizonDays)

	var dates []time.Time

	current := time.Date(from.Year(), from.Month(), 1, 0, 0, 0, 0, time.UTC)
	for current.Before(until) {
		lastDay := lastDateOfMonth(current.Year(), current.Month())
		if dayOfMonth <= lastDay {
			day := time.Date(current.Year(), current.Month(), dayOfMonth, 0, 0, 0, 0, time.UTC)
			if !day.Before(from) && !day.After(until) {
				dates = append(dates, day)
			}
		}
		current = current.AddDate(0, 1, 0)
	}

	return dates
}

func generateEvenOdd(from time.Time, even bool) []time.Time {
	until := from.AddDate(0, 0, generationHorizonDays)

	var dates []time.Time
	for day := from; !day.After(until); day = day.AddDate(0, 0, 1) {
		isEven := day.Day()%2 == 0
		if isEven == even {
			dates = append(dates, day)
		}
	}

	return dates
}

func lastDateOfMonth(year int, month time.Month) int {
	return time.Date(year, month+1, 0, 0, 0, 0, 0, time.UTC).Day()
}
